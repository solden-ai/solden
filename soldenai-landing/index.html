Routing decisions are rules. The model never gets a vote.

Match tolerance, sanctions, duplicates, budget gates, 3-way logic. All deterministic. Claude reads invoices and writes operator-facing prose; it doesn't decide anything routing-relevant. 

match   PO, GRN, invoice                 # tolerance 0.5% on amount
gate    sanctions(vendor)                # BLOCK on hit
gate    duplicate(invoice_no, vendor)    # BLOCK
gate    budget(gl, period)               # NEEDS_INFO if over
gate    bank_detail_change(vendor)       # NEEDS_INFO, verify

route   amount > 15_000                  : CFO
route   amount > 5_000                   : Controller
route   default                          : AP Lead

post    erp.bill                         # on approve, idempotent
log     audit_event                      # append-only, DB trigger

Finance complexity has outgrown its tools.

More vendors, more transactions, more systems. Coordination is still manual, fragmented, invisible. Three things shifted at once.

  1. 01 AI got reliable enough. Claude reads unstructured invoices, generates operator-facing prose, and answers operator queries. Five years ago this would have been brittle; today it's production-ready.
  2. 02 ERP buyers accept overlay tools. A decade of Ramp, BILL, and Tipalti softened the ground. Procurement no longer flinches at "sits beside your ERP." They expect it.
  3. 03 Approvals permanently moved to Slack and Teams. Remote work pushed finance decisions out of the ERP. They're not coming back. The tooling has to follow.
