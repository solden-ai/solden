Routing decisions are rules. The model never gets a vote.

Configurable match modes, tolerances, sanctions, duplicates, budget. All deterministic. The model reads. The model writes operator prose. The model never votes.

three_way_required

3-way matching required

PO, goods receipt, and invoice must all align. Missing GRN blocks the post and routes for review.

two_way_fallback

2-way fallback default

Try 3-way first. Accept PO + invoice when GRN is the only blocker. Price, currency, and quantity variance still gate.

policy_only

Approval policies only

Skip matching. Route every invoice through approval thresholds (amount band, vendor history, GL category).

Configurable per organization. Versioned: every match outcome is auditable back to the policy version it ran under.

# two_way_fallback (default mode)
match   invoice ↔ PO                     # tolerance 2% / amount floor 10
match   invoice ↔ GRN                    # if present; missing GRN → 2-way accept
gate    sanctions(vendor)                # BLOCK on hit
gate    duplicate(invoice_no, vendor)    # BLOCK
gate    budget(gl, period)               # NEEDS_INFO if over
gate    bank_detail_change(vendor)       # NEEDS_INFO, verify

route   amount > 15_000                  : CFO
route   amount > 5_000                   : Controller
route   default                          : AP Lead

post    erp.bill                         # on approve, idempotent
log     audit_event                      # append-only, DB trigger

Tamper-evident at the schema layer.

Append-only audit log enforced by database triggers, not application code. Every event hashes the prior event. Strip a row and the chain breaks. Your auditor sees the gap.

  • guarantee no row removable, no row backdated
  • enforced db_trigger before_update, before_delete
  • algo sha256(prev_hash || event)
  • retention 7 years, configurable per region

Finance complexity has outgrown its tools.

More vendors, more transactions, more systems. Coordination is still manual. Three things shifted.

  1. 01 The model got reliable. It reads unstructured invoices, writes operator prose, and answers queries. Brittle five years ago, production-ready now.
  2. 02 Procurement accepts overlay tools. A decade of Ramp, BILL, and Tipalti softened the ground. "Sits beside your ERP" is the expectation now.
  3. 03 Approvals moved to Slack and Teams permanently. Remote work pushed finance decisions out of the ERP. They're not coming back.

The model got reliable. The stack got open. The work didn't get coordinated. That gap is the bottleneck now.
